基于YOLOv8的饮料识别系统，能够检测10种常见饮料。

## 🚀 快速开始


如果你有GPU，运行完整训练脚本：
```bash

```

#### 💻 CPU训练
如果你只有CPU，可以使用以下选项：

**专门CPU训练脚本**:
```bash

```

**完整训练脚本** (自动检测GPU/CPU):
```bash

```

### 4. 测试模型

训练完成后，运行测试脚本：
```bash
python test_drink_detection.py
```

## 🤖 智能饮料贩卖机系统

### 功能特点

- **自动饮料检测**: 使用YOLOv8模型自动识别饮料种类和数量
- **智能数量统计**: 对比开门前后图片，计算购买差异
- **自动价格计算**: 根据预设价格自动计算消费金额
- **账单生成**: 自动生成详细的购买账单
- **交易历史**: 保存所有交易记录
- **批量处理**: 支持多组图片的批量处理

### 使用方法

#### 1. 运行智能贩卖机系统
```bash

```

```bash

```

1. **记录开门前状态**: 上传贩卖机装满饮料的图片
2. **记录关门后状态**: 上传顾客购买后的图片
3. **自动计算**: 系统自动计算购买情况和价格
4. **生成账单**: 自动生成详细的购买账单

## ⚙️ 训练参数说明

### 模型选择
- **YOLOv8n**: 最小模型，速度最快，适合实时检测和CPU训练
- **YOLOv8s**: 小模型，平衡速度和精度
- **YOLOv8m**: 中等模型，较好的精度
- **YOLOv8l**: 大模型，高精度
- **YOLOv8x**: 最大模型，最高精度

### GPU训练参数
- **epochs**: 训练轮数，建议100-300轮
- **batch_size**: 批次大小，根据GPU内存调整 (8-32)
- **imgsz**: 输入图片尺寸，默认640
- **patience**: 早停耐心值，防止过拟合

### CPU训练参数 (优化)
- **epochs**: 训练轮数，建议30-100轮
- **batch_size**: 批次大小，建议2-8 (根据内存调整)
- **imgsz**: 输入图片尺寸，建议320-416 (越小越快)
- **patience**: 早停耐心值，建议10-20

## 💻 CPU训练指南

### 为什么CPU训练较慢？
- CPU的并行计算能力比GPU弱
- 深度学习模型需要大量矩阵运算
- 内存带宽限制

### CPU训练优化建议
1. **使用较小的模型**: YOLOv8n 或 YOLOv8s
2. **减小批次大小**: 2-8 (根据内存调整)
3. **减小图片尺寸**: 320-416
4. **减少训练轮数**: 30-100轮
5. **关闭混合精度**: 自动处理
6. **减少数据加载线程**: 2个线程

### 预估训练时间
- **YOLOv8n + 50轮**: 约30-60分钟
- **YOLOv8s + 50轮**: 约60-120分钟
- **YOLOv8m + 50轮**: 约120-240分钟

## 🛠️ 故障排除

### 常见问题

1. **内存不足**
   - 减小batch_size
   - 减小图片尺寸
   - 使用更小的模型

2. **训练效果不佳**
   - 检查数据标注质量
   - 增加训练轮数
   - 调整学习率
   - 使用更大的模型

3. **类别不平衡**
   - 使用类别权重
   - 数据增强
   - 收集更多少数类别样本

4. **CPU训练太慢**
   - 使用YOLOv8n模型
   - 减小批次大小和图片尺寸
   - 减少训练轮数
   - 考虑使用云GPU服务

5. **智能贩卖机检测不准确**
   - 确保图片质量良好
   - 调整置信度阈值
   - 检查光照条件
   - 重新训练模型
